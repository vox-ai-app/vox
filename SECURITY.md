# Security Policy

## Supported versions

| Version  | Supported |
| -------- | --------- |
| >= 1.0.1 | Yes       |
| < 1.0.1  | No        |

## Reporting a vulnerability

**Do not open a public issue for security vulnerabilities.**

Report vulnerabilities privately via one of:

- **Email:** security@vox-ai.chat
- **GitHub private advisory:** [Report a vulnerability](https://github.com/info-arnav/vox/security/advisories/new)

Include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigations

We will acknowledge your report within 48 hours and aim to release a fix within 14 days for critical issues.

## Scope

In scope:

- The Vox Electron app
- Published npm packages (`@info-arnav/vox-*`)
- Remote code execution, privilege escalation, or data exfiltration vectors

Out of scope:

- Issues requiring physical access to an unlocked device
- Social engineering attacks
- Vulnerabilities in third-party dependencies (report directly to them, but let us know so we can update)

## Disclosure policy

We follow coordinated disclosure. Please give us reasonable time to patch before any public disclosure.
